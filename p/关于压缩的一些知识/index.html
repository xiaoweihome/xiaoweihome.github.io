什么是动态

动态范围(Dynamic Range)：指声音中最强和最弱信号之间的差异程度，表现为"大声和小声之间的区别"。

核心特征：

  • 动态≠音量：音量是绝对响度，动态是相对变化幅度

  • 大动态示例：唐僧念经时突然被惊吓大叫（从平稳到剧烈变化）（大动态带来夸张的表现力）

  • 小动态示例：持续均匀的念经声（音量恒定）（小动态能够让声音一致）

测量单位：分贝(dB)，指数级增长（每增加10dB相当于能量增加10倍）

什么是波形

波形(wave/envelope)：就是声音的形状，可以直观理解为声音的视觉化表现， 通过波形可以观察到声音的动态变化，但无法直接看出频率信息。

波形的四个重要部分：ADSR（Attack 启动 / Decay 衰变 / Sustain 延音 / Release 释放）

  • Attack: 声音从无到达到波峰的时间（如军鼓"啪"的瞬间）
  • Decay: 从波峰开始衰减的时间
  • Sustain: 声音持续阶段的电平保持
  • Release: 声音完全消失的过程

注意：这里说的 Attack 和 Release 与压缩插件中的是不同的，这里是形容波形

什么是压缩

压缩（Compressor）用于减少声音的动态范围，使声音更加平稳。

  • 通过设置合适的阈值和压缩比例，可以**把过大的声音压下来，让跳动的声音变得平稳，更易于人耳捕捉。**在录音过程中，可以限制非常大的动态声音到一个范围内，避免超出声卡范围

声音塑形

通过压缩来改变声音的音头和音尾，从而改变声音的特性。

  • 方法: 通过调整压缩的启动时间和释放时间，可以塑造出不同的声音形状。例如，延长军鼓音头或增大音尾，可以改变其打击感。

音染色彩

音染色彩是指压缩设备或插件在压缩过程中给声音带来的额外音色和音质变化。

  • 很多模拟硬件的压缩插件能带来音色和音染，这些设备通过不同的技术和设计，会给声音带来很多不一样的色彩。

例如，变压器可以给低频和中频带来饱满的色彩，电子元件可以带来谐波和音染。

压缩的各种参数

阈值（Threshold）

阈值是决定压缩是否启动的界线，超过阈值的信号会触发压缩，低于阈值的信号则不会。

阈值相对信号的高低会决定了压缩的幅度。

比例（Ratio）

决定了压缩的幅度。信号超过阈值后，动态降低的比例，每超过一定dB（如2dB），动态范围就会减少相应的dB（如1dB）。

如果设置二比一的比例，那么信号每超过阈值2dB，动态就减少1dB；

如果是四比一的比例，则每超过阈值4dB，动态减少2dB，以此类推。

高比例配合高阈值，低比例配合低阈值: 高比例（如8:1）配合高的阈值，压缩较重；低比例（如2:1）配合低的阈值，压缩较轻，声音更平滑。

拐点（Knee）

拐点的形状很大程度上决定压缩效果的特性。

  • 硬拐点（Hard Knee）: 拐点较硬，声音变化更硬朗，响度大。

  • 软拐点（Soft Knee）: 拐点较软，声音变化平滑、渐进、自然。

高比例搭配 Soft Knee 可以增加声音控制能力，使压缩效果更加自然，不会过度影响动态范围。

Soft Knee 优点: 提供了一种声音特别平滑、渐进、自然的感觉，增加了声音的控制力，不会使动态范围过度减小。

启动时间（Attack）

当信号超过阈值时，压缩器开始工作的时间间隔。

  • <5ms（快速）—— 5-20ms（中速）—— >30ms（慢速）

  • 1s = 1000ms = 1000000ps

  • 快速的启动时间可以很好的控制一些动态，特别快把声音尖的音头抓住，让声音的动态变得平滑、均匀、限制。

  • 但过快的启动时间会压制音头减少很多音乐的特性（比如吉他、打击乐、人声齿音）,压的过多就会缺少声音的质感，让声音感觉特别死，降低生动性。

  • 慢速的启动时间会保留强调声音的音头（很多声音的音头是它的特性之一），可以很容易识别到，增加更多细节和力量。

  • 但太慢了就会很容易错过，达不到控制动态的目的。

释放时间（Rlease）

决定压缩后信号恢复原始状态的时间。

  • <50ms（快速）—— 50-100ms（中速）> 100ms（慢速）

  • 快速的释放时间可以增加整体响度，提升延音，做一些特效的结果。

  • 但过快的释放时间可能会让声音过载或失真，因为它可能会再次触发压缩的机制，会出现不自然的 Pumping 效果（释放回来超过阈值又触发压缩）

  • 慢速的释放时间保证声音平稳，自然，更平稳的控制动态。

  • 但过慢的释放时间会把动态控制死，整体都是扁扁的感觉，没有律动。

  • 一般要在两个拍子之间释放完，比如军鼓第一下敲击完压缩后，在下一拍军鼓之前要释放完，如果压缩还在工作音量就会变小。

范围（Range）

限制最大增益衰减量（如最多压缩3dB），防止过度压缩破坏音乐动态。（更适合在总线或者母带处理阶段使用）

  • 母带处理阶段用于提升1-2dB响度时保护歌曲动态，如elysia设备的GR Limit功能

保持（Hold）

设置一个时间，在压缩启动后，等一下再开始释放。

  • 通过延迟释放来避免强烈的 Pumping 效果。

  • 用在鼓组上可以有效控制或者优化节奏感。

  • 可以用作混响的控制，防止人声间隙时混响突然反弹（设置100ms释放时间仍会出现），让混响保持一直在人声底下的感觉。

预判（Lookahead）

提前看到信号，提前做出反应。

  • 比 0ms 更快，负ms。

  • 有效避免超级快速压缩带来的音头爆裂感。

  • 虽然可以创造出干净的音色，但有些时候会缺少压缩带来的音染和特点。

压缩技巧

串联压缩

顾名思义，按顺序进多个压缩。

串联压缩的主要目的在于动态控制，每一个压缩插件进行相对微小且有针对性的处理。

常见顺序:

  • 第一级（瞬态控制）：快速启动时间（<1ms） + 快速释放时间

  • 处理对象：仅针对音频信号中最尖峰的瞬态部分（约1dB处理量）

  • 第二级（动态控制）：中速启动时间 + 快速释放时间

  • 处理对象：在第一级基础上控制下一层动态,让更多音头出来

  • 第三级（律动控制）：慢速启动时间 + 中速释放时间

  • 处理对象：整体声音的韵律感，提升小音量细节，让整个声音更平滑

  • 1176 与 LA-2A 两个压缩的搭配

（1176 抓住尖的地方，把它压下来，LA-2A 再做整体的动态。）

平行压缩

两个完全相同的音轨，一轨保持原样，另一轨进行压缩处理。

  1. 通过发送轨来实现：把原轨信号发送至压缩轨道。

  2. 原轨上实现：如果插件带 blend / mix 功能可以直接混合干湿信号。

平行压缩(parallel compression)，通过将原始信号与压缩处理信号混合实现，通过混合比例调整声音质感，做出一些特殊效果，而非主要控制动态。

  • 压缩参数可调得较夸张以突出音染效果。

  • 这个技巧一般不用于动态控制，而是为了通过压缩获得一种声音效果（如：底鼓轰轰声），所以在做之前需要清晰知道自己想要的声音。

侧链

侧链（Side——Chain）是指控制和触发压缩器工作的信号，压缩器只有“听到”侧链中的信号才会工作。

  • 内部侧链（Internal）：以当前音轨为目标做压缩处理。

（比如把压缩放进人声轨道，人声说话时，压缩检测到才开始工作。）

  • 外部侧链（External）：以其它音轨的信号为目标做压缩处理。

（比如：在背景音乐音轨上放压缩，以人声轨为目标，人声只要说话，压缩检测到信号开始工作，就会把背景音乐压下去。

  • 侧脸信号的强弱可以改变压缩的幅度。（增加压的就多，减少压的就少。）
  • 侧链中突出的频段，压缩会着重处理。

（如果想让某个频段压的多，就把这个频段在侧链里把它推起来，反之，如果不希望压缩压这个频段，就在侧链里面把这个频段减少。）

均衡和压缩的关系

先理解均衡与压缩的关系。

均衡放在压缩前

  1. 这时的均衡用于修复作用，把不需要的声音去掉后到达比较好的状态再进入压缩。

  2. 跟刚才侧链一样，用均衡来控制压缩着重的频段。（着重压缩的频段提高，想闪避压缩的频段降低）

（同时压的多的频段，也会有更多的音染色彩。）

均衡放在压缩后面

  1. 推多少dB就是多少dB的实际频段变化（如推3dB高频即得到3dB提升）

  2. 不会影响压缩器的工作状态

适用场景：

  • 压缩后的音色已满意，仅需微调频段平衡

  • 需要保持原始压缩特性不被EQ改变的情况

压缩家族的其它成员

多段压缩

可以在任何频段内点击一个位置进行压缩，例如只压缩2k到3k之间的频段，而不影响其他频段。

通过多段压缩，可以针对特定声音的频段进行调整，提供极大的控制能力。

去齿音

原理: 去齿音插件通过选择特定频段的点，并使用非常快速的启动时间和释放时间进行压缩，从而去除齿音。

  • 选择齿音频段（如女生通常在7.5k到12k左右），调整域值和range以控制去除齿音的程度。

  • 注意事项: 过度的压缩可能导致声音部分被吞没，因此通常将压缩量控制在四五个dB以内。

限幅

限幅器将声音限制在一个固定的幅度内，防止声音超过设定的天花板。

限幅器通常具有两个机制：

  1. 内部的 clipping 机制，硬生生的去除音头。（如果出现过多声音就不好听）

  2. 限幅机制，用于抓住超过限制的声音。

限幅器的启动时间和释放时间跟平常理解有些不同

  • 它的启动时间意思是你希望多长时间的声音放出去，让它撞到你的 clipping ，然后你的限幅尝试去抓它（比如启动时间 2ms，基本所有的音头都会撞到 cliping ）

  • 释放时间会直接影响整个歌曲的律动，需要很仔细的设计一个释放时间，释放时间长整体声音都会被压平，释放时间太短会不断触发 clipping 机制，导致过载。

  • 在限幅器中释放时间不能太长，不然声音就会被捏死；但太短出现过载也不好。
